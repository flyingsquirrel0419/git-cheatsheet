# Contributing

Thanks for improving this Git and GitHub cheat sheet.

## Good Contributions

- Correct a command, flag, or explanation.
- Add a scenario that developers commonly search for.
- Improve GitHub CLI or GitHub Actions coverage.
- Improve accessibility, responsiveness, or copy behavior in the website.

## Content Style

- Prefer scenario-first wording: "Undo last commit, keep changes" is better than "git reset".
- Include a short note explaining when to use the command.
- Mark destructive commands clearly.
- Keep examples generic and safe.

## Local Preview

Open `index.html` in a browser. No build step is required.

## Pull Requests

Use a focused PR with a short description of what changed and why.
